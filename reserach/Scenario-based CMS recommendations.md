### Executive Guide to CMS Categories Mentioned

To ensure decision-makers are speaking the same language, here is a brief glossary of the categories referenced in the table:

*   **Marketing-Centric SaaS:** All-in-one, closed-system platforms built specifically for marketers. *Examples: HubSpot CMS, Marketo.*
*   **Visual Builders:** Design-centric tools that output code but hide it from the user. *Examples: Webflow, Framer.*
*   **Hybrid SaaS:** Traditional monolithic architecture but modernized with APIs and better front-end performance. *Examples: Craft CMS, WordPress (with modern page builders).*
*   **Visual Headless:** Decoupled architecture (content via API) but features an in-app, visual, drag-and-drop page builder for marketers. *Examples: Storyblok, Builder.io.*
*   **API-First Headless:** Content is strictly treated as data. Requires a separate frontend (React/Next.js) to render. Highly developer-centric. *Examples: Contentful, Sanity.*
*   **Application Headless / BaaS:** Headless CMS combined with backend-as-a-service features (auth, database relations, logic). *Examples: Payload CMS, Xano.*
*   **Composable DXP:** A suite of best-of-breed tools (CMS, Commerce, Search) tied together via an orchestration layer, rather than one monolithic system. *Examples: Uniform, MACH architectures.*
